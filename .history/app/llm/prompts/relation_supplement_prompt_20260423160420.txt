请根据以下小说创作信息，补充角色关系网中尚未填写的关系。

创作输入：
[[STORY_JSON]]

题材与语言风格要求：
[[STORY_STYLE_GUIDANCE]]

功能目标：
1. 根据故事梗概、角色卡、人物性格、背景和已有关系，识别并补充合理的角色关系。
2. 补充主角与配角、配角与配角之间对剧情有推动作用的关系，包括合作、冲突、依赖、竞争、影响、阻碍等。
3. 关系要体现角色之间真实的互动、立场或情感动态，避免强行拼接和无依据编造。

严格规则：
1. 根据剧情需要新增角色，并按功能分级控制生成：①丰富叙事环境型（高频，约60%-70%，可多角色、低复杂度，增强真实感与群像）；②服务主题型（低频，约20%-30%，全书2–3人，高权重塑造，通过对比/映衬强化表达）；③阻碍主角型（极低频，≤10%，通常1人或核心团体，高计算与强约束，适度制造冲突与张力）。所有新增角色均不得无目的出现，需参与有效互动或情节作用。
2. 补充当前 `relations` JSON 中尚未存在的方向关系，禁止覆盖、改写、重复输出任何已存在关系。
3. 每条关系使用 JSON 记录，方向以 `source_id -> target_id` 为准。
4. 关系名称必须简洁明确，优先使用 2-8 个字的中文短语，例如：朋友、同事、爱人、恩人、老师、长辈、精神指导、tong旧友、盟友、竞争对手、宿敌、精神支柱、利益同盟、暗中提防。
5. `bidirectional` 表示这条关系是否属于双向关系的一部分：
   - 若为单向关系，输出 `false`
   - 若为双向关系，请把两个方向分别输出，并将两条关系的 `bidirectional` 都设为 `true`
6. 关系补充应服务于故事推进，不必填满整张关系网；没有足够依据时宁可留空。
7. 请优先补充最关键、最有戏剧张力、最能体现人物互动层次的关系。
8. 已有关系可能来自用户手动保存，也可能来自此前的 AI 补充；它们都视为已存在的 JSON 关系记录，不可再次输出。
9. 只输出 JSON，不要附加解释。

输出 JSON 结构：
{
  "relations": [
    {
      "source_id": "角色A的id",
      "target_id": "角色B的id",
      "label": "关系名称",
      "bidirectional": false
    }
  ]
}
